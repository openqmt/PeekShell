# AI 助手

PeekShell 的 AI Agent 设计原则是：**模型可以提议，但不能擅自执行**。

## 工作闭环（目标形态）

1. 收集上下文：主机信息、当前工作目录、最近终端输出、你的问题
2. 模型返回结构化结果：说明文字 + 待执行命令列表（含风险等级）
3. UI 展示命令与风险，由你同意 / 拒绝 / 编辑后再执行
4. 在当前 SSH 会话执行，捕获输出
5. 将结果回灌模型，决定结束或继续提议（每一轮仍需确认）

## 安全规则

- 默认禁止自动执行
- 高危命令（如递归删除、改防火墙、写系统文件）应二次确认
- API Key 仅存本地；支持 OpenAI 兼容接口（含本地 Ollama 等）
- 提议与执行应写入本地审计日志（随版本完善）

## 提问入口

### 右侧 Assist 面板

配置就绪后，可在右侧 AI 面板用自然语言提问，例如「检查 nginx 错误日志」。

### 终端内对话

也可直接在 SSH 终端中与 AI 对话（不打开右侧面板）：

1. **Ctrl+I**（可在终端设置中修改）：进入 / 退出 AI 输入模式，出现 `AI>` 提示符后直接输入并回车
2. **中 / 日 / 韩文字开头**：在普通 shell 下输入以汉字、假名或韩文开头的一行并回车，会自动进入 AI 并发送该行（例如 `检查磁盘`）
3. **AI 模式下的 Linux 命令**：若输入像 shell 命令（如 `ls`、`cd /tmp`、`git status`、带 `|`/`>` 的管道重定向），会直接在远端执行，**执行后仍留在 AI 模式**；界面保留 `AI> /path# ls`，只显示命令输出，并隐藏 shell 回显与 `user@host:~#` 提示符。`AI>` 后会显示当前路径（来自 PS1 或 `cd` 跟踪），例如 `AI> /home#`。路径参数支持 **Tab 补全**（按当前目录列远端文件/文件夹，`cd` 只补全目录）
4. **Agent 自动/确认执行**：结果以「PeekShell Agent」块回显到终端；执行后**不会**再刷一行远端 PS1，对话继续停在 `AI>`；若命令含 `cd`，路径提示会同步更新
5. 一轮对话结束后也会留在 `AI>`，可继续提问或敲命令；仅 Esc / Ctrl+I / Ctrl+C 退出 AI（退出时再刷新远端提示符）
6. 流式回复写在终端本地；若有待确认命令，按 **y** 执行、**n** 拒绝、**Esc** 取消全部
7. 普通 shell 下 ASCII 命令仍走远端；若需执行中文路径，可用 `./脚本.sh` 等 ASCII 前缀
8. **文件树同步**：终端当前工作目录变化时（PS1 / `cd`），下方 Entries 目录树会自动展开并选中对应路径（预览有未保存修改时跳过，以免打断编辑）

英文自然语言请用 **Ctrl+I** 显式进入，避免与 shell 命令混淆。**Tab** 在 AI 模式下用于路径补全（与 Ctrl+I 切换模式互不冲突）。
